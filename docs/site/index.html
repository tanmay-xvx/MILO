0x02Pipeline · english → silicon

One sentence becomes running machine code — through four gates

The host sends your request and the device's self-reported hardware manifest to the model. The generated Rust is wrapped in a no_std crate, compiled to a memory-capped wasm module, and streamed over MILO-Link. Before a single instruction runs, the receiver checks the frame size, an optional signature, and the import whitelist, then executes under fuel metering in a 64 KB sandbox.

"hold 3 m"→ GPT-4o→ Rust→ wasm · 64 KB→ MILO-Link→ 4 gates→ device→ telemetry

0x09Scenario A · hot-swap · demos/swarm_demo.py

A laptop flies five drones, retasks them mid-air, and repairs one — in 48 seconds

Five emulated quadcopters run one 943-byte controller that reads its altitude target from a live parameter slot: the echelon is data, not code. Mid-flight the whole formation is retasked with a 0.05 ms broadcast. Then a motor fault is injected — and the interesting part begins.

0x09 HOT_SWAP

The fault droops, the LLM ships the fix, the drone climbs back

A pure-proportional controller under a thrust deficit doesn't crash — it settles 1.9 m below its commanded altitude, exactly as control theory predicts. The host asks GPT-4o for a controller with integral action; 6.9 seconds later a 975-byte module is hot-swapped onto the running drone. It clears the import whitelist, boots in the sandbox, and flies the drone back to formation in 9.5 s. The other four never notice.

0x08Scenario B · autonomous cell · demos/factory_demo.py

A factory cell that diagnoses itself — and rewrites its own firmware

Three machines — a conveyor, a reflow oven on naive open-loop vendor firmware, and a pick arm — under one control station. Two incidents are injected without telling the controller. Both are handled autonomously, with GPT-4o as the supervisor and MILO as its hands.

0x0A SIGNED_PUSH

The heater drifts, so the LLM replaces the control law

When the oven's element starts running 35% hot, a new set-point wouldn't fix it — the firmware itself is wrong. The station catches 175.9 °C against a 165 °C limit, GPT-4o diagnoses the drift from telemetry, writes a closed-loop controller (835 B, 4.7 s), and pushes it between production cycles. The oven pulls back inside limits within one cycle and holds 148 °C for the rest of the shift — on hardware whose behavior changed under it.

A jam, diagnosed and cleared by another machine

Cycle 10: belt powered, zero items past the photo-sensor. The supervisor halts the belt, dispatches the arm for a clear-sweep (limit switches confirm travel), and restores production — 4.1 seconds from detection to recovery, throughput identical to the healthy baseline.

MCPAgent control plane · host/mcp

Any agent gets hands — through the Model Context Protocol

MILO ships a full MCP stdio server. Point Claude Code, Claude Desktop, or Cursor at it and the agent can stand up a fleet, read each device's manifest, write firmware to it in natural language, retask it live, and repair it — all through typed tool calls that pass the same four safety gates. This is the standard way for an LLM agent to actuate the physical world at the microcontroller level.

0x0ATrust model · SECURITY.md

The model is creative. The runtime is paranoid.

Nothing the LLM emits is trusted. Every code-loading opcode crosses four independent gates before a single instruction runs on hardware — and out-of-spec peripheral access is denied and logged, never silently ignored.

GATE 1 · bound

Frame cap

A payload length is checked against a 512 KB ceiling before any buffer is allocated. A hostile 4 GB length prefix is rejected, not honored.

GATE 2 · sign

Ed25519 signature

With a trusted key provisioned, only operator-signed modules run. require_signed refuses unsigned pushes outright. Verified on RISC-V and ARM, not just the host.

GATE 3 · scope

Import whitelist

The module's imports are parsed and checked against the 12-call Alphabet. Anything else — an unknown import, a WASI call — is rejected and never instantiated.

GATE 4 · sandbox

Sandbox + fuel

Execution is capped at 64 KB memory / 8 KB stack with per-instruction fuel. No pointer into firmware; an infinite loop exhausts gas and returns cleanly.

Capability denial is loud. A syscall aimed at a pin the manifest never advertised is a safe no-op that logs denied: pwm_set 7 not in manifest — visible to the host and the model, auditable, never silent.

0x01Board matrix · docs/PORTING.md

One driver corpus, every board — the compounding moat

The runtime, protocol, validation, signing, and syscall ABI are board-agnostic. A new board means implementing one trait and one entry point; it then inherits every driver ever written, and every new driver works on it. Board coverage × driver corpus is the product that compounds.

What "emulated" means, precisely. Every device in the demos is a separate milo-receiver process running the same Rust runtime, wasmi interpreter, syscall ABI, validation, signing, and framing that ship to ESP32-C3 and Pico hardware — only the peripheral layer is simulated, with physics models for thrust, heat, and belt motion so the model's code faces consequences.
